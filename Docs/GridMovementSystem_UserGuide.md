# 网格移动系统 (Grid Movement System) - 使用与配置指南

## 1. 系统概览 (这是什么？)

本系统是一套为 UE5 设计的**严格俯视角网格移动系统**，风格类似《双点医院》、《双点博物馆》或《过山车大亨》等老牌模拟经营游戏。

**核心特色：**
* **严格的棋盘格对齐**：无论角色怎么走，永远只会行走在尺寸固定的无形“网格（格子）”中心，绝不会站在两个格子中间。
* **物理材质感知**：不同的地面（如泥地、草地）可以设定不同的行走速度和通行消耗。
* **智能动态避障**：当玩家在游戏运行时放置建筑（障碍物）阻断了角色的去路时，角色会自动感知并重新规划绕行路线。
* **底层混合技术**：借用了 UE5 原生强大的 NavMesh（导航网格体）来进行寻路和遮挡计算，但屏蔽了原生的平滑物理移动，强行将结果映射成复古的“逐格跳跃”移动。

## 2. 核心文件结构与作用 (里面都有什么？)

对策划/测试人员来说，只需关注蓝图和数据资产（DataAsset），底层 C++ 代码已自动打理好一切。

### 2.1 核心驱动模块 (无需修改，只需知道存在)
* **`GridMapSubsystem` (网格地图子系统)**：整个系统的大脑。它记录了“哪一格能走、哪一格有泥地”等全部数据。游戏运行时自动存在。
* **`TerrainScanner` (地形扫描器)**：系统的眼睛。游戏开始时，它会从天空发射成千上万条射线扫射地面，根据坡度和障碍物自动生成网格数据。
* **`GridMovementComponent` (网格移动组件)**：角色的双腿。只要给任意的角色（Character）挂上这个组件，它就会剥夺原生的自由移动权，接管为严格的逐格移动。

### 2.2 策划需要配置的资产 (DataAsset)
* **`DA_GridConfig` (移动参数配置)**：
  * 控制基础移动速度（例如每秒走几格）。
  * 控制转弯速度。
  * 控制对角线移动规则（允许切角，还是必须直角转弯）。
* **`DA_GridMaterialRules` (地面材质规则)**：
  * 将 UE 的“物理材质（Physical Material）”翻译成网格属性。
  * 例如：配置“只要踩到 `PM_Mud (泥地)`，移动速度就变成 0.5 倍，且寻路权重翻倍”。
* **`DA_GridInputConfig` (输入配置)**：
  * 绑定鼠标左键/右键的点击事件（选中、移动、取消选中）。

### 2.3 测试用 Actor
* **`Bp_GridDebugActor` (调试器)**：把它拖进场景，它就会把隐形的网格画成五颜六色的线框显示出来（绿=可走，红=不可走，橙=太陡）。
* **`Bp_GridTestCharacter` (测试小人)**：挂载了网格移动组件的角色。自带一个 AI 脚本，运行后会满地图随机乱跑测试路径。

## 3. 编辑器环境配置指南 (如何搭建一个测试场景？)

如果你新建了一个空地图，需要按照以下步骤配置，网格系统才能正常工作：

### 步骤 1：开启 NavMesh 动态生成
系统依赖 NavMesh，并且需要它支持动态更新。
1. 点击顶部菜单栏 `Edit (编辑)` -> `Project Settings (项目设置)`。
2. 左侧搜索 `Navigation Mesh`。
3. 找到 **`Runtime Generation`**，将它改为 **`Dynamic`**。

### 步骤 2：铺设地形与 NavMesh
1. 在场景中放置你的地面模型（Plane 或 Landscape）。
2. **非常重要**：给你的所有地面模型添加一个 Actor Tag，名字叫做 **`GridTerrain`**。（如果不加这个标签，扫描器会无视这块地面，视为悬崖）。
3. 从 `Place Actors (放置Actor)` 面板拖入一个 **`NavMeshBoundsVolume`**，放大它，确保它包裹住你的整个活动地面。
4. 按键盘 **`P`** 键，确认地面上覆盖了绿色的可通行区域。

### 步骤 3：放置系统 Actor
1. 将 `Bp_GridDebugActor` 拖入场景。
   * 在它的 `Details (细节)` 面板中，你可以把 `Material Rules` 指向准备好的 `DA_GridMaterialRules`。
   * 勾选 `Debug Draw` 可以在屏幕上画出格子。
2. 将 `Bp_GridTestCharacter` (或者挂了网格移动组件的自定义角色) 拖入场景。
3. （可选）如果你需要点击控制功能，确保在 `World Settings (世界设置)` 或 `GameMode` 中，把 PlayerController 替换为 `Bp_GridTestPlayerController`。

### 步骤 4：配置特殊地面（如泥地/禁区）
1. 在内容浏览器创建物理材质（如 `PM_Mud`）。
2. 打开 `DA_GridMaterialRules` 资产，新增一条规则：
   * 选择 `PM_Mud`。
   * 设置 `Speed Multiplier` 为 0.5。
3. 打开你的地面材质球（Material），给特定的地面区域刷上对应的物理材质。
4. 运行游戏时，角色走到这片区域就会自动减速。

### 步骤 5：动态障碍物测试
游戏运行中，如果你想凭空“建造”一个不可穿越的机器/墙壁：
1. 放置一个模型。
2. 给它添加一个 **`NavModifierComponent`** 组件。
3. 在组件设置中，将 **`Area Class`** 设为 **`NavArea_Null`**。
4. 放置的瞬间，系统会自动检测到改变，底层的网格会变成红色（Blocked），且正在移动的角色会自动绕开它。
